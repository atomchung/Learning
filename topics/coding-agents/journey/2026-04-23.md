# 本週快照：我目前對 coding agent 設計權衡的理解

> 不是筆記，是「目前我認為的答案」。三個月後回看會知道自己在哪裡跳躍。

## 一句話當前理解
> 三家 coding agent 不是在競爭「誰最好」，而是在賭「agent 的主流工作形態會長成什麼樣」——Codex 賭非同步批量、Antigravity 賭多模態 IDE、Claude Code 賭即時人機協作。

## 本週有哪些跳躍
1. **從「比功能」到「比賭注」**：一開始我是在列三家的 feature 差異，這週才意識到功能差異背後是三種不同的世界觀押注。
2. **Harness 這個詞**：原本我以為這是技術名詞，但讀完 Phil Schmid 那篇才發現它其實是**產品戰略層級的概念**——harness 決定了你能做的所有決策的邊界。
3. **同模型不同 harness**：Opus 4.6 在 Antigravity 被壓到 1% thinking，這件事徹底改變了我「選工具要選模型」的直覺。

## 我還沒搞清楚的
- 多 agent 編排（Antigravity 的 Manager Agent）到底是噱頭還是未來？目前讀到的案例不夠多
- MCP 的實際擴展性有多強？還沒動手試過
- 「harness 是差異化戰場」這個判斷會不會太早——模型能力還在快速提升
