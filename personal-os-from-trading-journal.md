## 為什麼這件事對你有幫助

### 1. 你已經有「結構化分析」的肌肉

你寫 coding agent 比較文，是有定位、有維度、有結論的。同一塊肌肉拿來反思自己，門檻幾乎是零。

多數人卡在 Journaling 的點是「不會寫、寫不出深度」——你不會。

### 2. 投資模盤是 Journaling 的「最佳練習場」

模盤的價值不在「假裝賺錢」，在於**累積決策樣本但不付真錢學費**。

但如果不寫交易日誌，模盤的學習效率約等於零：
- 你會記得結果，記不得當初的判斷邏輯與情緒
- 一個月後復盤只能說「那次有賺/賠」，說不出「判斷錯在哪一步」

寫了之後會浮現的東西：

| 看見的東西 | 例子 |
|-----------|------|
| **重複錯誤模式** | 「我總在 -3% 加碼」「財報前都會手癢進場」 |
| **情緒 vs 決策耦合** | 哪些交易是焦慮驅動、哪些是分析驅動 |
| **個人 edge** | 哪類標的 / 哪類設置判斷準、哪類完全不準 |
| **規則 vs 直覺的勝率差** | 有 checklist 的交易 vs 衝動的交易，哪個賺多 |

這些東西**你不寫就永遠不知道**，看績效圖看不出來。

### 3. 你做模盤代表你在意「複利」

Journaling 的本質就是學習的複利。你已經接受了金錢複利的概念，自然會接受認知複利。

## 怎麼做：最小可行版本

**不要一次蓋大系統。先用兩週驗證會不會持續。**

### Step 1：建一個 trading journal（一筆交易一個檔案）

```
~/personal-os/trading/
├── 2026-04-26-TSLA-long.md
├── 2026-04-27-NVDA-short.md
└── reviews/
    └── 2026-04-week17.md
```

每筆交易模板（**進場時就寫**，不要事後補）：

```markdown
# TSLA Long @ 2026-04-26

## Setup
- 進場價 / 部位大小 / 停損 / 目標
- 為什麼進：技術 / 基本面 / 消息 / 模式
- Checklist 通過了嗎？(打勾)

## Mental
- 進場時情緒（1–5 焦慮 / 興奮 / 平靜）
- 是否有 FOMO？是否有報復心態？

## Hypothesis
- 我預期接下來會發生什麼
- 多久內、什麼訊號代表我錯了

## Exit (出場時補)
- 出場價 / 報酬 / 時長
- 出場原因：停損 / 達標 / 手癢 / 看錯

## Postmortem (當天結束補)
- 預測對了嗎？對在哪 / 錯在哪
- 重複了哪個過去的錯誤？
- 學到一件事：
```

### Step 2：每週日 30 分鐘做交易週回顧

固定模板，只回答四個問題：

1. 這週**勝率**和**盈虧比**是多少？
2. 最好的一筆交易做對了什麼？
3. 最爛的一筆是什麼**模式**——是新錯誤還是重複錯誤？
4. 下週唯一要改的一件事是什麼？

### Step 3（兩週後再考慮）：把 AI 接進來

兩週後你會有 ~20 筆紀錄。這時餵給 Claude 問：

- 「列出我這兩週重複出現的進場理由 / 出場理由」
- 「我哪類情緒下的交易勝率最低？」
- 「找出我自我描述為『手癢』的所有交易，共通點是什麼？」

這時 AI 才有素材可以幫你看到自己看不到的 pattern。

> **先不要顛倒順序**——AI 沒有原始資料就只能講廢話。

---

## 為什麼這個切入點對你特別合適

- **有立即回報**：交易日誌直接改善模盤決策品質，不像一般 Journaling 要三個月才看得到效果
- **有客觀錨點**：交易有勝率、有報酬，比情緒日記更容易看出「寫了 vs 沒寫」的差別
- **練成之後可遷移**：一旦你習慣對交易做這套，搬到「職涯決策 / 人際決策 / 健康決策」會非常自然
- **跟你會的東西接得上**：你已經會寫結構化分析，這就是把同樣手法用在自己身上

## 下一步選項

**A. 直接動手** — 在 repo 裡建好 `trading/` 目錄、模板檔、一個 `new-trade` 的 CLI script，你下一筆模盤就用

**B. 先聊清楚** — 你告訴我模盤的標的類型（美股 / 台股 / 加密 / 期權？）、目前持續多久、最大痛點是什麼，再把模板調得更貼

**C. 繼續走完模組介紹** — 把剩下的（習慣 / 健康 / 財務 / CRM / 閱讀 / 目標）看完，再回頭做整體規劃

---

## 重要原則（別忘了）

1. **先持續兩週再加功能**——多數系統死在第二週
2. **進場時寫，不要事後補**——事後寫會合理化決策，失去 journal 的意義
3. **不要追求完美模板**——丑模板用兩個月，比美模板用兩天有用 100 倍
4. **AI 是回顧助手，不是寫作助手**——你要先有原始素材
5. **隱私底線**：交易紀錄牽涉財務細節，建議純本地 + git encrypted，不要進雲端 SaaS
