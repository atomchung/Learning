# YC Demo Day 近四年（2022-2026）創業方向研究

> 整理日期：2026-05-03
> 範圍：YC W22 → W26（含 X25 / F25），對照組為同期非 YC 但定義整代的 AI 公司
> 視角：PM 結構化分析，給 ting 評估自己的 build 方向

## TL;DR

四年走完三個階段：**通用 SaaS（2022）→ Generative AI 應用（2023）→ AI Agents 全面接管（2024-2026）**。AI 占比從 S22 的 **15%** 一路漲到 W26 的 **~100%**，且重點從「AI 加值在現有產品上」→「Agent infrastructure + Vertical Agent 直接取代人力職務」。Demo Day 成績紀錄被連續打破：W26 有 **14 家公司 Demo Day 當天就到 $1M ARR**，史上最高。

## 3. 跨年差異：四個關鍵維度

### 維度 1：AI 占比與「AI 含義」演變
```
2022: 15% AI（多數是 ML 應用，例如電腦視覺、推薦）
2023: 34→60% AI（多數是 GPT wrapper、prompt 應用）
2024: 50→67% AI（Agent 出現，Copilot 普及）
2025: 60→70% AI（Agent 變主流；voice agent、vertical agent 興起）
2026: ~100% AI（"AI 是基線不是論點"——競爭點轉到下一層）
```
**含義變化**：早年是「我們在產品裡加了 AI」，現在是「我們的 agent 能直接做完一個職務」。差別是工作流是否還需要人。

### 維度 2：主題類別轉移
| 期間 | 火紅類別 |
|------|---------|
| 2022 | B2B SaaS / Fintech / Consumer App / Healthcare API |
| 2023 H1 | GenAI 應用、DevOps copilot |
| 2023 H2 - 2024 | AI 寫程式、AI 銷售、AI 客服、AI 法律、AI 醫療（Cognition + Devin 帶起 vertical agent） |
| 2025 | Agent infrastructure（observability/memory/security）+ "Cursor for X"（每個職能都有 Cursor 對應的 vertical agent） |
| 2026 | **Physical world**：Robotics、Energy、Agriculture、Aerospace、Construction；Healthcare（drug discovery、autonomous primary care） |

> 觀察：軟體職務「被 AI 吃完」後，YC 把資源往 atom（物理世界） push。

### 維度 3：估值與成長速度
| 期間 | 典型 Demo Day 估值 | 領頭羊估值 |
|------|------------------|------------|
| W22 / S22 | $10-20M post | 沒特別跳的 |
| W23 / S23 | $20-30M post | 數家百萬 ARR |
| W24 / S24 | $30-50M post | Cognition 從 $0 → $2B / 1 個月 |
| W25 / X25 | **$30M baseline，$100M+ for 公司有 $1M+ ARR** | Cursor 從 $0 → $2B ARR / 3 年（史上最快 SaaS） |
| S25 / F25 | $100M+ 募資成常態 | Mercor $2B → $10B（8 個月） |
| W26 | 14 家 Demo Day 當天 $1M ARR | 不公開但 batch 整體可能是 YC 史上最強 |

### 維度 4：團隊規模與創辦人特徵
- **2022**：典型 3-5 人團隊；技術背景 + MBA
- **2024-2026**：**< 10 人達 $10M ARR** 已不稀奇；25% YC 公司「95% 程式碼由 AI 寫」
- **F25**：**18-25 歲創辦人**比例顯著上升（學生跳過大學直接做）
- **W26**：年輕創辦人 + AI 工程槓桿 = 比資深團隊跑得更快

依 README.md 列的「最可能 build 方向」，對照這份趨勢：

| 中文版 OpenClaw（LINE/WeChat） | 全球 YC 在做 vertical agent，**沒人做台灣/繁中** | **最值得做**——空白市場，技術門檻你能 cover |
| 垂直 Agent Workflow（投資/內容） | YC 2025-2026 的火力點正在這 | 必須拿 distribution（你的 personal_os + investment_note 內容是優勢） |
| 瀏覽器自動化 | Browser Use 已開源，紅海 | 改做「對接特定行業的 browser agent」（例如台灣銀行/政府網站） |
| AutoResearch 變體 | 學術 niche，有空間 | 中文研究 + 領域知識堆疊（例：中文金融研究 agent） |

**最關鍵的時序訊號**：
1. **2026 batch 已往物理世界 push**——軟體 agent 已飽和，台灣有硬體製造優勢，這個方向長期會變熱
2. **「Cursor for X」紅利 2024-2025 結束**——再做通用 vertical agent 已晚；**做地理 niche（繁中、台灣監管場景）才是窗口**
3. **YC 偏好 < 10 人 + 高 AI 槓桿**——一個人 build 完全合理，不需要找 cofounder 才能起步

## 5. 延伸研究：Cursor for X 增長與 K 字分歧（2026.05）

> 核心問題：**模型越強（GPT-5.5、Claude 4.7、DeepSeek V4），vertical agent 增長還能持續嗎？**
> 結論：**不是減速，是 K 字分歧**——頂級玩家加速，thin wrapper 大量死亡。

### 5.2 為什麼模型強化反而幫到 vertical agent

| 時點 | 事件 | 對 vertical agent 影響 |
|------|------|----------------------|
| 2024 H1 | GPT-4 / Claude 3.5 普及 | Thin wrapper 大量誕生 |
| 2024 H2 - 2025 | Thin wrapper **大規模死亡**（"mass extinction"） | Cursor / Harvey / Sierra 開始壓制 |
| 2025 Q4 | Claude 4.7、GPT-5 | Vertical agent 進化到 long-horizon 任務 |
| **2026.04** | **DeepSeek V4-Pro 開源、價格 = GPT-5.5 的 1/7** | 模型層商品化加速，**vertical agent 毛利反而上升** |
| 2026 H1 | Sierra ARR 破 $150M、Harvey 破 $190M、Cursor 破 $2B | 贏家加速 |

四個機制：

1. **Moat 不在模型，在 context**：Harvey 的 moat 是 1300+ 律所工作流嵌入，不是 GPT；Sierra 的 moat 是 100+ 企業客服歷史
2. **模型差距只有 6 個月壽命**：純靠 base model 領先打競爭的，半年內被拉平
3. **模型變便宜 = vertical 公司毛利上升**：DeepSeek V4-Pro = GPT-5.5 的 1/7 成本，但對企業客戶收費不變
4. **OpenAI / Anthropic 進不來深 vertical**：他們做 horizontal（ChatGPT、Operator、Claude Code），但「美國刑事辯護律師專用 + PACER 整合 + Westlaw 嵌入」這層深度需要 domain sales、合規、整合

### 5.3 K 字分歧的判定標準

| 訊號 | 跑出來 | 會死 |
|------|--------|------|
| **Context moat** | 有獨家業務數據、工作流嵌入（Harvey 的律所文件、Sierra 的客服 log） | 純調用 OpenAI API、產品就是 prompt + UI |
| **Net retention / churn** | NRR > 120%、客戶用越久越深 | Churn > 30%、客戶 3 個月就走（11x.ai） |
| **Distribution / 通路** | 有行業圈子、有 vendor 整合（Clay 的 GTM 圈、Cursor 的開發者社群） | 只能 cold outbound、沒有自然口碑 |

### 5.4 對 ting 的補充意涵

**好消息**：
- **vertical agent 紅利還沒結束**——Harvey 558% YoY、Sierra 5.7x YoY 都是 2025-2026 的數字
- 個人 build 在這個時代是真機會：模型成本降 7 倍、context moat 比模型 moat 大
- ting 的 `personal_os` / `investment_note` 內容資產正好是 distribution moat 的雛形

**壞消息**：
- **「Cursor for 一個英文世界已有的職務」窗口已關**——Harvey 占了法律、Sierra/Decagon 占了客服、Cursor/Cognition 占了 coding
- 想做美國市場的通用 vertical agent，已經要打 $10B 估值的對手

**真正的窗口**：
1. **繁中 / 台灣監管 / 台灣行業 SaaS** — 國際巨頭不做的 niche
2. **Context moat 來自 ting 的內容資產**（投資寫作、xhs 研究法）— 把 distribution 內建
3. **不要做「AI for 已被 SaaS 解的問題」**，要做「AI 才能解的問題」（自動化研究而非 dashboard、agent 替代分析師而非提供工具）
