# Learning Repository

個人學習紀錄，讓 Claude Code 成為我的學習夥伴。

## 這個 repo 的目的

1. **記住我的偏好** — 輸出格式、語言風格、我在乎的角度
2. **累積學習脈絡** — 每次學新主題時，Claude Code 能讀到我過去學過什麼
3. **串聯知識** — 自動把相關主題連起來，幫我 connect the dots

## 我的偏好

### 輸出格式
- 語言：繁體中文
- 用**表格**做比較分析（我是 PM，習慣看結構化對比）
- 用 **bold** 標重點，不要一大段散文
- 先給結論再展開，我可以決定要不要深入
- 每篇筆記最後附「閃卡」方便我日後快速複習
- 適當使用 `>` blockquote 標注核心洞察

### 我的背景
- 產品經理（PM）視角，關注商業邏輯、用戶價值、競爭格局
- 對技術有興趣但不是工程師，喜歡理解「為什麼這樣設計」而非實作細節
- 偏好用類比解釋抽象概念

## 筆記結構

所有學習筆記放在 `notes/` 目錄，每個 `.md` 檔開頭有 frontmatter：

```yaml
---
title: "筆記標題"
date: "2026-04-05"
tags: ["tag1", "tag2"]
related: ["other-note-slug"]   # 手動標記相關筆記
summary: "一句話摘要"
flashcards:
  - q: "問題"
    a: "答案"
---
```

### 標籤系統
標籤用來串聯主題。新增筆記時：
1. 先看 `notes/` 裡現有筆記的 tags，盡量重用
2. 如果是新領域才建新 tag
3. 在 frontmatter 的 `related` 欄位手動連結強相關的筆記

### 知識索引
`notes/INDEX.md` 是所有筆記的索引，按標籤分類，方便快速瀏覽。

## 新增學習筆記的流程

當我提出一個學習主題時，Claude Code 應該：

1. **先讀 `notes/` 目錄**，了解我過去學了什麼
2. **找出關聯**，看看新主題跟哪些舊筆記有交集
3. **整理輸出**，用我偏好的格式寫成筆記
4. **標注連結**，在新筆記裡引用相關舊筆記，也回去更新舊筆記的 `related` 欄位
5. **更新索引** `notes/INDEX.md`
6. **Commit & push**

## App（可選）

`app/` 目錄有一個 Next.js 手機優先的 Web App，用來瀏覽筆記和複習閃卡。
這是可選的，核心價值在 `notes/` 裡的 markdown 內容本身。
