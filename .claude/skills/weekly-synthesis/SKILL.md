# weekly-synthesis：讓知識複利的內容層 review

睡前步驟是**增量**的——每次只看一條 inbox。這個 skill 補上缺的**批次** pass：定期掃整週，主動抽出單條看不出來的跨主題模式。對照 meta-review：

- **meta-review = harness 層**（讓系統複利）：讀 defects，改規則本身。
- **weekly-synthesis = 內容層**（讓知識複利）：讀 inbox/notes，抽判斷、結算預測。

兩個都要，但別混。這個 skill 不碰規則，只碰知識。

## 為什麼存在

攝取流程的第 4、5 關（決策時調出 / 被證偽後更新）靠單條睡前步驟做不到——要批次回看才看得出模式、才結算得了預測。這 30 分鐘同時完成「重組」和「校準」兩件只有回看才能做的事。詳見 `notes/claude-code-second-brain-noah-brier.md`。

## 步驟

1. **掃輸入**：`inbox.md` 最近約 7 天 + `git log --since` 看 `notes/` 這週改動的檔 + 本週對話訊號。不複述流水帳，目標是抽 pattern。
2. **抽跨主題模式**：找這週**反覆出現 / 可互相印證 / 互相矛盾**的判斷。重點是「單條看不出、合起來才看出」的東西（例：三個不同主題都指向 harness>model）。
3. **結算到期預測**：讀 `profile.md`「你當前開放的疑問」裡 `check:` 日期 ≤ 本週的，逐條問「現實怎麼判了？」標 **✅ 對 / ❌ 錯 / ⚪ 無結論**。**錯的特別寫一句「我當時為什麼錯」**——這句是模型校準的燃料，不能省。沒到期的略過。
4. **標過期卡**：grep `topics/**/cards/` 的 `freshness:`，列出已過期的，提醒「當時快照、不當現在事實」。
5. **提卡片升級（≤3，不硬湊）**：這週的判斷裡，哪些夠元、跨脈絡（你會想填 `appears-on`）→ 提議升級成 `topics/` 卡。預設留在筆記層，只有真能重用的才升。
6. **交報告 + 掌舵**：把 2–5 攤給用戶。用 AskUserQuestion 或直接問，讓用戶**糾正抓錯的、決定哪些畢業成卡、結算預測對錯**。Claude 抽、用戶判。
7. **落地**：用戶定後更新 `profile.md`（預測結算、話題浮動、開放疑問增刪）、建卡、commit、merge 進 main。

## 邊界

- 不是每天跑，**週五一次**或用戶喊才跑。可用 `/loop 7d /weekly-synthesis` 掛，但預設手動（雲端 session 是 ephemeral，排程不可靠）。
- 抽 pattern，不複述流水帳。**這週沒跨主題訊號就直說**「這週是單點積累、無 pattern」，別硬湊。
- 反膨脹：升級卡 ≤3、不硬湊；多數判斷的歸宿是筆記層。
- 預測結算要誠實標對錯，尤其錯的——這個 skill 的價值一半在「逼自己認當時錯在哪」。

## 預測帳怎麼讀（跟 profile.md 的約定）

`profile.md` 的「開放疑問」每條帶 `check:YYYY-MM`，就是預測帳的到期日。週綜合掃到到期的就結算：

- `✅ 對` / `❌ 錯` / `⚪ 無結論` + 一句話
- 錯的補「為什麼錯」，對的可順手指向沉澱的卡片
- 結算完的可留痕（標 ✅/❌ 不刪），或移進「已沉澱的判斷」段
