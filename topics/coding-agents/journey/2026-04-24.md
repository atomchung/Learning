# 結構跳躍日：把 compare-coding-agents.md 回推成 13 張原子卡

> 不是筆記，是「目前我認為的答案」。這一篇不是因為時間到而寫，而是因為發生了一次「整體結構重整」，值得獨立記錄。

## 一句話當前理解
> 我原本以為「長文是學習成果」，今天發現**長文其實是學習副產品，真正的資產是背後那堆可重用的原子判斷**。從 13 張卡反推 compare-coding-agents.md 可以重寫三次、角度全不同，而反向做只能重寫一次。

## 這次跳躍的內容
1. **卡片的 title 形式**：從「harness 介紹」改成「harness 的影響大於換模型」——title 變成**判斷句**而非**主題名**，標題本身就是可用知識。
2. **關係類型化**：不再只寫「相關連結」，而是標示 `← 支持 / ↔ 對比 / → 引出`。這逼我說清楚為什麼相關。
3. **appears-on 多脈絡**：一張卡同時屬於多個白板，這件事改變了我對「歸檔」的直覺——知識沒有單一歸屬。
4. **起點卡是磁鐵不是目錄**：`_start.md` 列的不是預設章節，而是「已拆 / 累積中 / 待拆」三欄——結構從卡累積到一定密度後自己浮現。

## 我還沒搞清楚的
- 卡片多到一定程度會不會**反而難檢索**？13 張還好，130 張呢？
- 「待拆的卡」那個清單會不會變成一個不斷膨脹的債務？什麼時候該刪？
- Journey 要寫多細？今天這篇算「跳躍日」細一點，週五那篇該更簡短嗎？
- 卡之間的連結要雙向維護嗎？A 寫了「→ B」，B 要不要寫「← A」？

## 對比上週（2026-04-23）
上週的 journey 說「不在乎定價了」。這週的頓悟更深一層：**連長文本身都只是副產物**。這是個明顯的抽象層級跳躍——上週我在重新排列資訊優先級，這週我在重新定義「什麼才是產出」。
